### quick-dataflow

Description: A focused exercise exploring React data flow — how state is owned by a parent component and shared with children through props, and how children communicate back up to the parent through function props.
Concepts covered:

State ownership — why state lives in the parent
Passing data down to child components via props
Passing functions down as props so children can trigger state changes in the parent
Controlled inputs — tying an input's value to React state
Local state in a child component to hold temporary values before sending up
Live update pattern vs. form submit pattern
Multiple components consuming the same piece of state simultaneously

That's honest, accurate, and reads well to anyone looking at your GitHub — including a future interviewer scanning your repos.
